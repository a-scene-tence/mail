# 디자인 (design.md) — 29cm 정통 미니멀

> 이 문서는 작업하며 지속 업데이트한다. 참고 디자인(Framer 링크)은 봇 접근이 막혀 직접 확인하지 못해, 29cm 에디토리얼 톤을 기준으로 정의했다. 실제 스크린샷 확보 시 값을 보정한다.

## 1. 디자인 원칙

- **에디토리얼 미니멀**: 흑백 중심, 큰 타이포, 넉넉한 여백, 1px 헤어라인으로 정보 위계.
- **절제**: 그림자·둥근모서리·색을 최소화. 콘텐츠(메일)가 주인공.
- **정렬**: 좌측 정렬, 그리드 기반, 일관된 리듬.

## 2. 컬러 토큰

| 토큰 | 값 | 용도 |
|---|---|---|
| `--ink` | `#0A0A0A` | 기본 텍스트, 강조 |
| `--paper` | `#FFFFFF` | 기본 배경 |
| `--paper-off` | `#FAFAFA` | 보조 배경/섹션 |
| `--gray` | `#6B6B6B` | 보조 텍스트 |
| `--hairline` | `#E5E5E5` | 1px 구분선/보더 |
| `--accent` | `#1A1A1A` | (절제) 버튼/포커스. 필요 시에만 |

> 포인트 컬러는 거의 쓰지 않는다. 상태색(읽지 않음 등)은 굵기/점(dot)으로 표현.

## 3. 타이포그래피

- **본문/UI**: Pretendard (system fallback 포함).
- **디스플레이 강조**: 세리프(헤드라인·섹션 타이틀의 에디토리얼 강조). 없으면 Pretendard로 폴백.
- 위계
  - Display: 40–56px / 좁은 자간(-0.02em) / 굵게
  - Title: 24–32px
  - Body: 15–16px / line-height 1.6
  - Label(소제목/메타): 11–12px / 대문자 / letter-spacing 0.12em / 그레이

## 4. 레이아웃 / 스페이싱

- 12-컬럼 그리드, 최대 콘텐츠 폭 720–960px(WebView는 모바일 우선 1컬럼).
- 스페이싱 스케일: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64.
- 섹션 구분은 그림자 대신 **헤어라인 + 여백**.

## 5. 컴포넌트 톤

- **Button**: 보더/언더라인 중심, 채움은 잉크 블랙. 라운드 최소(2–4px).
- **Hairline**: 1px `--hairline` 구분선 컴포넌트.
- **Label**: 대문자 + 넓은 자간 메타 라벨.
- **TextField**: 밑줄(underline) 스타일 입력, 포커스 시 잉크 블랙 밑줄.
- **ProviderCard / MailListItem**: 헤어라인으로 구분된 리스트 행, 호버 시 미묘한 배경 페이드.

## 6. 화면 와이어 (개요)

- **로그인(`/login`)**: 상단 큰 타이틀 → 제공자 카드 리스트(Gmail/네이버/다음/Outlook) → 선택 시 OAuth 리다이렉트 또는 IMAP 입력 폼.
- **홈/계정(`/`)**: 등록된 계정 목록 + "계정 추가" 진입. 계정 없으면 빈 상태 안내.
- **메일 목록(`/mail`)**: 상단 계정 필터(라벨) → 헤어라인 구분 리스트(발신자/제목/스니펫/날짜, 읽지 않음 dot). 비어있으면 스켈레톤/빈 상태.
- **읽기/작성(이후)**: 본문 단일 컬럼, 작성은 from 선택 + 제목/본문 + 발송.

## 7. WebView / 반응형 고려

- 모바일 1컬럼 우선, 안전영역(safe-area) 패딩, 큰 터치 타깃(44px+).
- 앱인토스 뒤로가기/가시성 브리지는 `lib/apps-in-toss.ts`에서 분기(웹에서는 no-op).
