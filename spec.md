# 기획서 (spec.md) — 통합 이메일 클라이언트

> 이 문서는 작업하며 지속 업데이트한다. 변경 시 "변경 이력"에 한 줄 남긴다.

## 1. 제품 개요

흩어져 있는 여러 이메일 계정(Gmail, 네이버, 다음 등)을 **한 곳에서 로그인해 읽고 보낼 수 있는** 통합 메일 앱.

- **개발/테스트**: Vercel(웹)
- **배포**: 앱인토스(App-in-Toss) 미니앱 (Toss 앱 내 WebView)
- **디자인 톤**: 29cm 정통 미니멀(에디토리얼)

## 2. 타깃 / 문제

- 업무·개인·쇼핑 등 목적별로 계정이 분산되어, 앱을 옮겨 다니며 확인해야 하는 번거로움.
- 하나의 단정한 화면에서 여러 계정의 메일을 모아 읽고 답장한다.

## 3. 핵심 사용자 시나리오

1. **계정 추가**: 제공자(Gmail/네이버/다음/Outlook…)를 고르고 로그인 → 계정이 목록에 추가된다.
2. **메일 읽기**: 통합 받은편지함 또는 계정별로 메일 목록을 보고, 항목을 열어 본문을 읽는다.
3. **메일 보내기**: 작성 화면에서 보낸 계정(from)을 고르고 발송한다.

## 4. 기능 범위

### MVP
- 다중 제공자 등록(제공자 레지스트리 기반)
- 로그인(OAuth: Gmail / IMAP·SMTP: 네이버·다음 등 앱 비밀번호)
- 메일 목록 조회, 본문 읽기, 발송
- 통합 받은편지함 + 계정별 필터

### 이후
- 검색, 라벨/폴더, 첨부파일, 푸시 알림(앱인토스 브리지), 답장/전달, 임시보관함

## 5. 다중 제공자(Provider) 모델

사용자가 서비스를 고르고 로그인만으로 메일을 불러온다. 제공자는 **레지스트리**로 추상화한다.

- `auth: 'oauth'` — Gmail/Google Workspace (refresh token, 비밀번호 저장 불필요)
- `auth: 'imap'` — 네이버/다음/Outlook 등 (IMAP/SMTP + 앱 비밀번호)
- 공통 인터페이스 `MailGateway`(list/get/send)로 화면은 제공자를 몰라도 동작.

자세한 타입: `lib/providers/types.ts`.

## 6. 아키텍처

앱인토스는 **SSR 금지 + 정적(static export) 번들**을 CDN에 업로드한다. 서버 로직은 번들에서 못 돌리므로 둘로 분리한다.

```
프론트엔드(정적 export, Next output:'export')  ──HTTPS──▶  메일 백엔드(Vercel 서버리스 /api/*)
  · Vercel 웹 + 앱인토스 WebView 공용                       · OAuth2 / Gmail API
                                                           · IMAP/SMTP (imapflow / nodemailer)
```

- 하나의 Vercel 배포가 정적 프론트(`out/`)와 `/api/*` 함수를 호스팅.
- 앱인토스 번들은 정적 프론트만 포함하고 같은 Vercel 도메인의 `/api/*`를 호출.
- 자격증명(refresh token, IMAP 비밀번호)은 클라이언트 저장 금지 → 서버에서 암호화 보관(저장소는 다음 단계: Vercel KV/Postgres).

## 7. 보안 요구사항

- 자격증명은 절대 클라이언트(localStorage/번들)에 저장하지 않는다.
- OAuth refresh token / IMAP 비밀번호는 서버에서 대칭키로 암호화 저장.
- 토큰 전송은 HTTPS + `Authorization` 헤더(쿠키는 WebView 제약 고려).
- 비밀/키는 환경변수로만(`.env.example` 참고), 레포에 커밋 금지.

## 8. 앱인토스 배포 흐름 (요약)

1. 프론트를 `output:'export'`로 정적 빌드(`out/`).
2. 앱인토스 빌드 툴체인(`@apps-in-toss/web-framework`)으로 번들 생성.
3. 앱인토스 콘솔 업로드 → 심사 → Toss CDN 배포.
4. 파일 기반 라우팅: `intoss://{appName}/path` 매핑.

## 9. 마일스톤

- **M1 (현재)**: 기획 문서 3종 + 프로젝트 골격(정적 export 빌드 통과, 로그인/메일 빈 화면, 제공자 레지스트리·게이트웨이 스텁).
- **M2**: Gmail OAuth 실연동(로그인 → 목록 → 읽기).
- **M3**: IMAP 제공자(네이버/다음) 연동 + 발송.
- **M4**: 통합 받은편지함, 검색, 첨부.
- **M5**: 앱인토스 콘솔 업로드/심사/배포.
